Rss Reader

The website is a basic RSS reader that lets users enter an RSS feed URL and obtain content. HTML, CSS, and JavaScript generate a header, main content, and footer for the website. The header has an RSS feed URL input box and an article fetching button. The main content section will dynamically add RSS feed items. Creative Commons licence images and licence details are at the bottom. XMLHttpRequest, a JavaScript object, sends GET queries to the RSS2JSON API, which transforms RSS feeds to JSON. The JavaScript function parses the JSON response and dynamically produces DOM components for each item, including title and description elements. The primary content section receives the produced pieces. CSS styles the header, main text, and footer with little modification. A top navigation bar, centred main text, and centred footer make the website simple and tidy. For layout, CSS uses width, padding, and text alignment. JavaScript event listeners manage user interactions like clicking the "Get Articles" button to show RSS feed items on the page. JavaScript displays "The request failed; please check your RSS URL" in the main content area whenever a request fails.

Reflection

Learning CSS, HTML, and JavaScript for the first time, as well as gaining a better grasp of the Document Object Model (DOM), and JavaScript event handling, has been an eye-opening trip for me as a commerce student specialising in IT and IS.After six weeks of intensive study, my opinion of web development's potential and flexibility has changed significantly. I started out by learning the fundamentals of CSS and HTML, which allowed me to organise and create web pages with a focus on aesthetics. Having the ability to make dynamic and interactive online content using CSS was fascinating to me. As I learned more about JavaScript, I was impressed by the power it gave web sites in terms of functionality and engagement. I learned the fundamentals of JavaScript syntax and programming concepts including variables, data types, operators, and control flow.

The notion of the DOM, though, is what introduced me to the fluidity of web programming. It was a game-changer for me to learn that the Document Object Model (DOM) specifies the structure of an HTML document and allows for the manipulation of its elements using JavaScript. I gained knowledge on how to dynamically update web content by choosing and editing items, navigating the DOM tree, and more. The ability to record and react to events like button clicks, form submissions, and key strokes was another amazing idea I learned while working with JavaScript.

I learned how to apply these theories in the actual world through coding exercises and projects. I am now capable of making dynamic web sites with flexible designs and interesting user experiences. Along the journey, I realised the significance of debugging and troubleshooting as I overcame obstacles and improved from my mistakes.

Even as a commerce student, I recognise the need of knowing how to build websites in the modern world. A company's online presence and interactions with customers can benefit substantially from its employees' abilities to design, develop, and manage websites and web apps. Additionally, I see the possibility for businesses to leverage JavaScript and the DOM to create tailor-made solutions like e-commerce websites, interactive forms, and data visualisations. In sum, it has been both difficult and gratifying to learn CSS, HTML, JavaScript, and the Document Object Model (DOM) for the first time. I'm excited to go deeper into web development and use what I've learned in my future studies and career.
